This document is intended to provided information guiding the installation and development process, ensuring that all system requirements are met. The following entities may find the document useful:

2.       Concept of Operations

`Wi-Fi-based Indoor Position System (WIPS)’ aims  to track the position changes of mobile facilities based on the Wi-Fi finger print detection.  It will facilitate its users to position the mobile device through the Wi-Fi fingerprints. WIPS has one UI  to register a mobile device,  one UI to import the information of Wi-Fi hotspots and one UI to import maps on which Wi-Fi hotspots and registered devices are shown. For more details on the usage and capabilities of the application read the section,System Capabilities.

2.2 System capabilities

Wi-Fi-based Indoor Position System is a software project to track the position changes of mobile facilities based on the Wi-Fi finger print detection. Its abbreviation is WIPS. The system will run on a mobile platform, such as Android. WIPS has a UI to register a mobile device and import the information of Wi-Fi hotspots. When a user wants to use the system, he must first register his device in the system.

After the user completes the device registration, the system will import the Wi-Fi hotspot information. When a mobile device tries to connect to a Wi-Fi hotspot, information exchange occurs, which is called Wi-Fi fingerprint. The system will collect the Wi-Fi fingerprint information. If there is an error in the work, the system will not proceed to the next step, but will try to repeat the work until it is completed successfully.

The system can calculate the location of the device according to the Wi-Fi fingerprint. If the device is in the building, the system will draw the internal structure of the building, and then use the Wi-Fi hotspots inside the building to calculate the location of the device. The location information of the device will be marked on the map of the system. After calculating the location information, the system will save the information, so that users can query the history at any time.

All functions are guaranteed to run correctly by the system administrator. The system administrator will monitor the data information of various functions in the background. The whole system will be perfect, it has reliable algorithm and excellent background staff. Even if there are errors, they will be solved in the shortest time.

4.1 Module: algorithm module--#include “all.h”

In the module of algorithm, we aim to enable WIPS to give a specific location of users. In this module, we need to read the wifi information storage path of each map from the database module. We need wifi information file from the module of Server.And besides, we rely on the module of Server to provide us MAC address and the signal strength of wifi selected from current position. In this way, we use all the information to give the specific location of users to the module of server. The positioning accuracy is about 5 meters

 

4.2 Wi-Fi Fingerprint - Pigeon Farm

The use of Wi-Fi to estimate location is a great approach, since Wi-Fi Access Points(AP) are readily available in large quantities in today's indoor environment and it is possible to use available mobile devices on the user's side.

The key part of the whole process is the Wi-Fi Fingerprint acquisition - the module we are working on. Based on the information we obtain, specific algorithm will be able to calculate the accurate position of the user.

The following picture clearly shows modules which are related to Wi-Fi Fingerprint and the relationship between them.

Aims of Module Wi-Fi Fingerprint are as follows:

1.       Provide an interface for Module Client. Whenever a user position requirement is activated, get Wi-Fi Fingerprint information and return it, so that Module Client will be able to send it to Module Algorithm.

2.       Provide an interface for Module Server. Whenever the admin requires the current Wi-Fi Fingerprint,  get Wi-Fi Fingerprint information back to Module Server.

7.       Glossary

Source

Source means where the data come from in each use case.

Administrator’s responsibility

Administrator ought to collect enough WIFI fingerprints, which will be used in device location. Also, he needs to draw the inner structure of a building, and save it as inner map data. Besides, it is his duty to make sure the system running normally.

Map adjustment

When administrator finishes drawing the inner structure of a building, he needs to adjust the map to make it look better. When he feels the preview of the map is nice, he will choose to save the map.

Map data

-Inner structure:  It is drawn by administrator of the application.

-Outer structure :The shape of the building.Map outside the building may not be shown.

Query position

Query position means that users can use the software to know the position of the devices currently.

Historical position

Historical position means that users can use the software to know the position of the devices at a certain time in the past.

Register

Register is the first necessary thing users should do before using the software.

Wi-fi fingerprint

Because Wifi is ubiquitous in all buildings of interest for this application, we will implement the positioning system based on this technology. Therefore, because the exact position of all the Wifi Access Points is not known, we will use a technique named Wifi fingerprinting as a location estimation technique. The administrator is responsible for the management of the fingerprint data collection.
